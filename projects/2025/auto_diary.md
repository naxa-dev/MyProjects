## Auto_diary (자동 일지 생성 웹앱)

### 개요
하루 동안의 활동 데이터를 수집(크롤링/로그)하고, 이를 정리·요약해 **자동 일지**를 생성/관리하는 개인 프로젝트입니다.  

### 흐름도

> 사용자/스케줄러 → 크롤링 (구글 로그인 → 내 활동 접속 → 크롤링) → LLM(GPT API) → Discord

사용 영상 : https://www.youtube.com/shorts/ikSM-owaP-o

---

### Tech Stack
- **Language:** Python
- **Web:** Flask
- **UI:** Jinja2 Templates (`templates/`) + Static assets (`static/`)
- **Crawling/Automation:** Selenium
- **Notification:** Discord Webhook (옵션)
- **Storage:** SQLite / 파일 저장 / DB 연동(선택)
- **Packaging (옵션):** PyInstaller (EXE 빌드)

---

### 주요 기능 (Functional)
#### 1) 활동 데이터 수집(크롤링)
- 브라우저 활동/기록 페이지(예: 방문 기록/검색 기록/시청 기록 등)에서 **일자별 활동 로그 수집**
- 로그인 세션/쿠키 기반 수집 흐름 지원(옵션)
- 수집 실패/차단 대응을 위한 기본 예외 처리 및 재시도(옵션)

#### 2) 데이터 정제/분류
- 수집된 로그에서 불필요한 텍스트 제거(정규식/클린징)
- 사이트/카테고리별 분류(업무/학습/엔터/쇼핑 등)
- 중복 제거 및 시간순 정렬

#### 3) 자동 일지 생성(요약/템플릿)
- 하루 로그를 기반으로 **일지 템플릿** 생성  
  - 오늘 한 일 시간별 요약
  - 인상 깊었던 항목
  - 내일 할 일/회고 포인트(옵션)

#### 4) 디스코드 알림 연동(Discord Webhook)
- 일지 생성 완료 시 디스코드로 요약 결과 자동 전송
- 실패/오류 발생 시 에러 로그 간단 알림(옵션)
- “오늘의 일지 링크/핵심 요약” 형태로 메시지 포맷 지원

#### 5) 스케줄링/자동화(옵션)
- 특정 시간에 자동 수집 → 자동 생성 → 자동 알림
- 간단한 cron/Task Scheduler 연동 또는 Python 스케줄러 적용 가능
